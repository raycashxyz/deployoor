# Hardhat

deployoor reads Hardhat `artifacts/` — **both Hardhat 2 and Hardhat 3** — and generates typed deployers from them.

## Compile and generate

```bash
npx hardhat compile && npx deployoor generate
```

| Input              | Location           |
| ------------------ | ------------------ |
| Compiled artifacts | `artifacts/`       |
| Config file        | `hardhat.config.*` |

## Auto-generate on compile

Add `@deployoor/hardhat` to skip the separate `generate` step:

```bash
pnpm add -D @deployoor/hardhat deployoor viem
```

```js
// hardhat.config.js
require("@deployoor/hardhat");

module.exports = { solidity: "0.8.24" };
```

Every `hardhat compile` regenerates `./deployers/`:

```bash
npx hardhat compile
# deployoor: generated 3 deployer file(s)
```

## Hardhat v3

Supported by `deployoor generate` — the reader resolves each artifact's inline `buildInfoId` to `artifacts/build-info/<id>.json` (Hardhat 3's layout). The `@deployoor/hardhat` auto-generate plugin above targets Hardhat 2's task API, so on Hardhat 3 run `deployoor generate` explicitly after `hardhat compile`:

```bash
npx hardhat compile && npx deployoor generate
```

Runnable examples: [`examples/hardhat`](https://github.com/raycashxyz/deployoor/tree/main/examples/hardhat) (Hardhat 2, auto-generate) and [`examples/hardhat-v3`](https://github.com/raycashxyz/deployoor/tree/main/examples/hardhat-v3) (Hardhat 3, explicit generate) — compile → generate → deploy → committed record.
